# Glossary

> **This page is `source: analysis`.** Definitions are compiled from across the wiki and made consistent here. The conceptual sources are cited (each term points to its concept page); the definitions themselves are this project's wording.

Terms used throughout this wiki, in one place.

## Core terms

### LLM-wiki

A persistent, interlinked markdown knowledge base maintained by an LLM. See [[core-idea]].

### Raw source

A file in `raw/` containing input material the user has curated. Immutable to the LLM. See [[layer-raw-sources]].

### Wiki page

A markdown file in `wiki/` written and maintained by the LLM. See [[layer-wiki]].

### Schema

The configuration file (`AGENTS.md` in this project) that defines conventions for how the wiki is structured. See [[layer-schema]].

### Concept page

A wiki page whose `type` is `concept` — an idea, term, or pattern (e.g., [[ingest-pipeline]]). The most common page type.

### Entity page

A wiki page whose `type` is `entity` — a named thing: a person, tool, paper, dataset.

### Summary page

A wiki page whose `type` is `summary` — a recap of one specific raw source. Typically named after the source.

### Analysis page

A wiki page whose `type` is `analysis` — interpretation that goes beyond what the raw sources literally say. Must be visibly marked as such in the body. Examples: this page, [[implicit-constraints]], [[open-questions]].

### Navigation page

A wiki page whose `type` is `navigation` — an index or TOC. Currently only [[index]].

## Operations

### Ingest

The operation that processes a raw source into wiki pages. See [[operation-ingest]] and [[ingest-pipeline]] for the seven steps.

### Query

The operation of asking questions against the wiki. See [[operation-query]].

### Lint

The operation that health-checks the wiki — finds contradictions, orphans, broken links, gaps. See [[operation-lint]].

### Query-as-write loop

The mechanism by which queries that exceed the wiki's coverage trigger web search + promote the new knowledge back into wiki pages. The second engine of compounding (alongside ingest). See [[query-as-write-loop]].

## Conventions

### Wiki link

`[[kebab-case-page-name]]` — a textual convention pointing to `wiki/kebab-case-page-name.md`. Resolved by the LLM, not by any viewer.

### Frontmatter

The YAML block at the top of every wiki page and raw source. Contains `title`, `type`, `source`, `updated`, `tags` for wiki pages; `source_url`, `source_type`, `fetched_at`, `ingested_hash`, etc. for raw sources.

### Source attribution (inline)

`(source: raw/<file>#<anchor>)` — used inline in wiki page bodies to point any non-trivial claim back to the raw source it came from.

### Source tag (frontmatter)

`source: video | analysis | external | mixed` in a wiki page's frontmatter. Marks the provenance of the page's content. See [[source-attribution]].

### Delta detection (ingest)

The mechanism `/wiki-ingest` uses to skip raw files that haven't changed: it computes the current body hash and compares to `ingested_hash` in the raw's frontmatter.

### Promote (query)

`/wiki-ask`'s default behavior of writing the new knowledge from a web-searched answer into the wiki as new or updated pages. Disabled with `--no-promote`.

### Notability heuristic

The LLM's judgment about whether a web-searched answer is worth promoting. Current heuristic: introduces a new term, makes a new connection, or cites a new external source.

## File layout

### `raw/`

Directory of immutable raw sources. See [[layer-raw-sources]].

### `wiki/`

Directory of LLM-owned wiki pages. See [[layer-wiki]].

### `.claude/commands/`

Directory where Claude Code reads slash command definitions. Each file (e.g., `wiki-init.md`) becomes a slash command (e.g., `/wiki-init`).

### `AGENTS.md`

The canonical schema file. See [[layer-schema]].

### `log.md`

Append-only log of every `/wiki-ingest`, `/wiki-ask` promotion, and `/wiki-lint --apply` operation.

### `README.md`

Install + quickstart for human users.
